\chapter{\ifenglish Conclusions and Discussions\else บทสรุปและข้อเสนอแนะ\fi}

\section{\ifenglish Conclusions\else สรุปผลการดำเนินโครงงาน\fi}
โครงงานการพัฒนาแพลตฟอร์มพาณิชย์อิเล็กทรอนิกส์ "Elder Store" มีวัตถุประสงค์หลักเพื่อสร้างระบบนิเวศทางเทคโนโลยีที่ช่วยลดช่องว่างความเหลื่อมล้ำทางดิจิทัล (Digital Divide) \cite{wai_older_users} และยกระดับคุณภาพชีวิตของผู้สูงอายุให้สามารถเข้าถึงสินค้าและบริการที่จำเป็นได้อย่างสะดวกรวดเร็วและปลอดภัย โดยคณะผู้จัดทำได้ดำเนินการออกแบบและพัฒนาระบบตามหลักการอารยสถาปัตย์ (Inclusive Design) \cite{inclusive2022} และก้าวข้ามมาตรฐานระดับโลกของสมาคมผู้บกพร่องทางสรีรวิทยา (WCAG 2.1) \cite{wcag2023} ควบคู่ไปกับการใช้กลุ่มสแตกเทคโนโลยีสมัยใหม่ ได้แก่ React.js, Next.js และ PostgreSQL เพื่อเสริมสร้างเสถียรภาพความเร็วและสถาปัตยกรรมความมั่นคงความปลอดภัยในการทำธุรกรรมสูงสุด \cite{pressman2014}

จากขั้นตอนการทดสอบและประเมินผลโดยผู้เชี่ยวชาญเชิงเทคนิค (บริษัท Iconnex Thailand) และกลุ่มตัวอย่างผู้ใช้งานจริง (End-User Testing) ผลลัพธ์เชิงประจักษ์แสดงให้เห็นว่า ระบบแพลตฟอร์มสามารถทำงานได้บรรลุตามข้อกำหนดความต้องการทางวิศวกรรมทุกประการ กลุ่มผู้สูงอายุที่เป็นประชากรเป้าหมายหลักมีความพึงพอใจต่อหน้าจอส่วนติดต่อผู้ใช้งาน (UI) ที่มีตัวอักษรขนาดใหญ่ สะอาดตา และไม่มีวงจรแบนเนอร์สิ่งรบกวนตา (Distraction-free) \cite{krug2000} ส่งผลให้สามารถรับภาระข้อมูล เรียนรู้ และปฏิบัติการสั่งซื้อสินค้าตั้งแต่ต้นจนจบกระบวนการหน้าต่างชำระเงินโอนบิลได้ด้วยตนเอง โดยไม่ต้องอาศัยความหยิบยื่นความช่วยเหลือจากบุคคลรอบข้าง ซึ่งสอดคล้องกับเจตนารมณ์ของโครงงานในการปลูกฝังความรู้สึกพึ่งพาตัวเองและรู้สึกเห็นคุณค่าในศักยภาพตนเอง (Self-Reliance \& Self-Esteem) ของประชากรในยุคสังคมผู้สูงอายุอย่างเด่นชัด

\section{\ifenglish Challenges\else ปัญหา อุปสรรค และแนวทางการแก้ไข\fi}
ตลอดระยะเวลาการดำเนินงานภายใต้วงจรการพัฒนาซอฟต์แวร์แบบว่องไว (Agile SDLC) \cite{beck2001} คณะผู้จัดทำได้เผชิญกับข้อจำกัดและอุปสรรคทางโครงสร้างวิศวกรรมทางข้อมูลบางประการ ซึ่งได้มีการนำมาอภิปรายวิเคราะห์และวางวิธีการแก้ไขจุดบกพร่อง (Troubleshooting) ไปตามแต่ละรอบสปรินต์ของการทำงาน ดังนี้:

\begin{enumerate}
  \item \textbf{ข้อจำกัดด้านบูรณภาพการกรอกข้อมูลของผู้ประกอบการ (Data Integrity Constraints):}  
  \textit{ปัญหาลักษณะนี้:} ในช่วงเริ่มต้นของการทดสอบระบบวงจรหลังบ้าน (Admin Dashboard) พบว่าผู้ประกอบการมักจะละเลยการระบุข้อมูลสินค้าอัปโหลดที่สำคัญในบางช่อง เช่น คุณสมบัติเฉพาะองค์ประกอบ หรือคำอธิบายรูปภาพสำหรับผู้รับการช่วยเหลือทางสายตา (Alt-text) ซึ่งส่งผลกระทบโดยตรงต่อระบบการสืบค้นหาข้อมูลและละเมิดมาตรฐานข้อบังคับเครื่องมือผู้อ่านหน้าจอ WCAG \cite{wcag22_2023}  
  \textit{แนวทางแก้ไขเชิงเทคนิค:} ทีมผู้พัฒนาขจัดปัญหาโดยเขียนชุดเงื่อนไขโมดูลตรรกะควบคุมรูปแบบฟอร์มล่วงหน้า (Form Validation) โดยบังคับให้ฟิลด์องค์ประกอบที่ส่งผลต่อคุณภาพสถาปัตยกรรมเว็บไซต์เป็นช่องที่ "บังคับระบุข้อมูล" (Required Fields) ก่อนจะอนุญาตให้เส้นทางเทอร์มินัล API ทำการอนุมัติพุชบันทึกลงระบบโครงสร้างตารางฐานข้อมูลเชิงสัมพันธ์ พร้อมจัดทำตัวป๊อปอัปให้คำแนะนำ (Tooltip) คอยพยุงวิธีการพิมพ์อักษรคีย์เวิร์ดที่ถูกต้องควบคู่กันไป

  \item \textbf{ความท้าทายในตรรกะการกำหนดพื้นที่เป้าหมายการสัมผัส (Elderly Touch Target Challenges):}  
  \textit{ปัญหาลักษณะนี้:} ในการทำการบ้านประเมินทดสอบประสบการณ์เชิงลึกแบบตัวต่อตัว (Usability Testing) \cite{nielsen2000} พบว่าจุดโฟกัสที่ผู้พัฒนาประเมินด้วยตาเปล่าว่ามีขนาดใหญ่พอแล้ว ยังคงเป็นอุปสรรคทางคณิตศาสตร์ต่อผู้สูงอายุบางราย ระยะห่างระหว่างขอบปุ่มกดที่ชิดกันเกินไปทำให้เกิดปฏิกิริยาอาการ "นิ้วลั่น" (Fat Finger Error) ไปโดนคำสั่งลบตะกร้าสินค้าโดยไม่ได้มีเจตนาตามการศึกษาทางการยศาสตร์ผู้สูงอายุ \cite{accessibility_elderly_2012}
  \textit{แนวทางแก้ไขเชิงเทคนิค:} คณะผู้จัดทำรื้อและจัดเรียงโครงสร้างแนวตั้งระบบกริดตาข่าย (Grid Layout System) ใน Tailwind CSS ใหม่ทั้งหมด \cite{tailwind2023} โดยขยายวงรัสมีแกนเป้าหมายสัมผัส (Tappable Area) ให้กว้างขั้นต่ำทะลุเกณฑ์เป็น 48x48 พิกเซล \cite{mobileux2021} และผลักค่าแนวกันชนพื้นที่ว่างอากาศ (Margin/Padding) ให้ห่างกันมากขึ้น พร้อมสร้างสคริปต์สกัดกั้นหน้าต่างข้อความเตือนภัย \textit{"ยืนยันว่าคุณต้องการกระทำพฤติกรรมนี้?"} (Confirmation Dialogs) ก่อนส่งสเตทสถานะที่ทำลายโครงสร้างทิ้งทุกครั้ง

  \item \textbf{ข้อจำกัดด้านปริมาณเปอร์เซ็นต์กลุ่มตัวอย่างประเมินการออกแบบ (Sample Size Limitation):}  
  \textit{ปัญหาลักษณะนี้:} ในกระบวนการทดสอบการเป็นที่ยอมรับทางตลาด (UAT) ถูกทำภายใต้สภาวะแรงกดดันด้านข้อจำกัดเวลาทุนสร้างโครงงาน ส่งผลให้อัตราผลตอบรับรีพอร์ตมาจากเครือข่ายกลุ่มเครือญาติเป็นหลัก ซึ่งมีโอกาสที่จะเกิดความเอนเอียงทางความระลึกใจ (Systematic Bias) ในการเทคะแนนความพึงพอใจ 
  \textit{แนวทางแก้ไขเชิงอุดมคติ:} น้อมรับสถานะข้อจำกัดดังกล่าวไว้เป็นข้อสังเกตเพื่อพัฒนารุ่นต่อไป (Iterative Release) และได้วางมาร์คพิมพ์เขียวเอกสารร่างชุดข้อกำหนดเพื่อนำสคริปต์แพลตฟอร์มออกทดสอบประเมินสเกลใหญ่ในเชิงปริมาณทางสถิติ กับกลุ่มประชากรสุ่มสเปซอวกาศที่มีความหลากหลายภูมิหลัง (Diversified Demographics) ขยายวงรันสเกลทดสอบเบต้าก่อนที่จะพับลิชโดเมนเนมสู่แสงสว่างระบบเซิร์ฟเวอร์ผลิตภาพ (Production Server) อย่างเป็นทางการเพื่อผู้ด้อยโอกาส
\end{enumerate}

\section{\ifenglish Suggestions and further improvements\else ข้อเสนอแนะและแนวทางการพัฒนาต่อยอด\fi}
แม้โครงงาน Elder Store ฉบับต้นแบบ (Minimum Viable Product) จะประสบความสำเร็จตามไกด์ไลน์ที่ตีกรอบไว้ แต่ด้วยวงการเทคโนโลยีที่หมุนและมีแพตเทิร์นวิวัฒนาการอย่างก้าวกระโดด คณะผู้จัดทำจึงส่งมอบชุดข้อเสนอแนะรวบรวมไว้เป็นไอเดียศึกษาและปรับปรุงต่อยอดระบบคอมไพล์เลอร์ในรันไทม์อนาคต ดังต่อไปนี้:

\begin{itemize}
  \item \textbf{การพัฒนาซิงก์ผสานอัลกอริทึมปัญญาประดิษฐ์ (AI/ML Integration):} 
  ควรศึกษาขยายผลโอกาสบูตแคมป์บูรณาการระบบกลไกการเรียนรู้ของเครื่อง (Machine Learning) แฝงผูกติดเข้าไปสังเกตการณ์ระบบบันทึกพฤติกรรมการคลิกเลือกของฐานข้อมูลลูกค้า เพื่อเทรนนิงโมเดลอัลกอริทึม "กล่องตัวช่วยแนะนำสินค้าอัจฉริยะ" (Smart Recommendation System) ที่เรียนรู้นิสัยและอ่านใจแม่นยำสูงในระดับปักเจกบุคคล (Personalized Algorithm) พร้อมดึงสถาปัตยกรรม AI ประยุกต์วิเคราะห์พยากรณ์คาดเดาความน่าจะเป็น เพื่อเสริฟชุดเมนูวิตามินหรือสินค้าอุปโภคที่ผู้สูงอายุดูจ้องกำลังจะต้องการขึ้นมาดักรอก่อนโดยไม่ต้องลงพิมพ์ค้าหาด้วยตนเอง

  \item \textbf{การสร้างแพคเกจแอปพลิเคชันข้ามระบบปฏิบัติการพื้นเมืองแบบสแตนด์อะโลน (Cross-Platform Native Application):} 
  รากฐานโครงสร้างของชุดสถาปัตยกรรมด้านการจัดขนถ่ายพอร์ต API ใน Backend ปัจจุบันมีความเหนียวแน่นยืดหยุ่นสูง ออกแบบมาเพื่อเป็นไมโครเซอร์วิส (Microservices Concept) สามารถลอกมาผูกต่อยอดควบคู่กับเฟรมเวิร์กจัดการแอปพลิเคชันอุปกรณ์สมาร์ตโฟน เช่น React Native หรือ Flutter ย้ายมวลมิติจากหน้าจอเบราว์เซอร์ (Web Application) สู่แอปพลิเคชันรันโปรแกรมติดตั้งฝังลึกบนสโตร์หน่วยความจำ (Native App) โดยสมบูรณ์แบบ ซึ่งจะปลดล็อกใบอนุญาตเข้าถึงฮาร์ดแวร์ระบบการโทรศัพท์ฉุกเฉิน เซนเซอร์อ่านลายนิ้วมือ และระบบปฏิทินยิงการแจ้งเตือนพุช (Push Notifications) เอาไว้สะกิดคอยเตือนให้ชำระบิลประวัติ หรือแจ้งส่งเสียงกระตุ้นให้ทานยา/เวชภัณฑ์แร่ธาตุอาหารเสริมได้ทรงเกียรติและตรงเวลามากขึ้นกว่าขีดจำกัดเบราว์เซอร์

  \item \textbf{ระบบสั่งการคำนวณรหัสด้วยเสียงและแชตบอทปัญญาประดิษฐ์เฝ้าไข้ (Voice Assistant \& Conversational AI):} 
  ควรลงทุนตั้งคลัสเตอร์ประเมินและผนวกรวมเอนจิ้นระบบค้นหาสินค้าด้วยเจตนาทิศทางเส้นเสียงพูดสั่งงาน (Voice Search Optimization) เพื่อทลายช่องว่างกำแพงข้อจำกัดความเสื่อมถอยรอยต่อในการพิมพ์สัมผัสบนคีย์บอร์ดที่ล่าช้าของผู้สูงอายุ \cite{uxdesign2023} พร้อมปูรากฐานบอตปัญญาประดิษฐ์ซ้อนทับรองรับการวิเคราะห์พิมพ์ตอบแชทด้วยการประมวลผลภาษาบรรยายธรรมชาติ (Natural Language Processing: NLP) จำลองการมีลูกหลานเสมือนจริงคอยดูแลนั่งแนะนำหน้าร้านค้าอยู่พรีเมี่ยมเคียงข้างบนหน้าจออย่างเป็นมิตรตลอด 24 ชั่วโมง ซึ่งถือเป็นสถิติใหม่และจิ๊กซอว์ก้าวสำคัญพลิกโฉมหน้าปะวัติศาสตร์เทคโนโลยีเชิงสุขภาวะ (Health Technology) บนบรรทัดฐานสังคมผู้สูงวัยรูปแบบใหม่ของมนุษยชาติ
\end{itemize}
