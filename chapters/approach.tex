ในบทนี้จะอธิบายถึงรายละเอียดสถาปัตยกรรมโครงสร้างและการกำหนดวิธีการดำเนินงาน (Methodology) สำหรับการพัฒนาแพลตฟอร์มพาณิชย์อิเล็กทรอนิกส์ "Elder Store" โดยกระบวนการทั้งหมดถูกออกแบบภายใต้แนวคิดผู้ใช้เป็นศูนย์กลาง (User-Centered Design) เพื่อให้มั่นใจว่าเทคโนโลยีและฟังก์ชันการทำงานที่สร้างขึ้น จะสามารถรองรับข้อจำกัดและตอบสนองต่อพฤติกรรมการใช้งานของผู้สูงอายุได้อย่างมีประสิทธิภาพสูงสุด

\section{ภาพรวมของระบบ (System Overview)}
สถาปัตยกรรมของระบบ Elder Store ได้รับการออกแบบโดยแบ่งแยกหน้าที่การทำงานออกจากกันอย่างชัดเจน (Separation of Concerns) ระหว่างส่วนการแสดงผลและส่วนการประมวลผล เพื่อให้ระบบสามารถรองรับผู้ใช้งานจำนวนมากและง่ายต่อการบำรุงรักษาในอนาคต โดยประกอบด้วย 2 ส่วนหลักหลัก ดังนี้:
\begin{itemize}
  \item \textbf{ส่วนติดต่อผู้ใช้งาน (Frontend):} ดำเนินการพัฒนาด้วยไลบรารี \textbf{React.js} ร่วมกับเฟรมเวิร์กจัดการสไตล์ \textbf{Tailwind CSS} เพื่อสร้างหน้าเอกสารเว็บแบบตอบสนอง (Responsive Single Page Application) สาเหตุที่ใช้เทคโนโลยีนี้เพื่อให้ผู้พัฒนาสามารถปรับแต่งขนาดตัวอักษร ความเปรียบต่างของสี และการจัดวางองค์ประกอบให้เรียบง่าย สะอาดตา และสอดคล้องกับหลักอารยสถาปัตย์ (Universal Design) สอดรับกับภาวะถดถอยทางการมองเห็นของผู้สูงอายุ
  \item \textbf{ส่วนประมวลผลและการจัดการหลังบ้าน (Backend):} ขับเคลื่อนด้วยเฟรมเวิร์ก \textbf{Next.js} ที่ทำงานบนพื้นฐานของ Node.js ซึ่งมอบประสิทธิภาพในการทำ Server-Side Rendering (SSR) ช่วยโหลดหน้าเว็บได้รวดเร็วขึ้น ทำงานร่วมกับระบบจัดการฐานข้อมูลเชิงสัมพันธ์ \textbf{PostgreSQL} เพื่อรับรองความถูกต้องปลอดภัยของข้อมูล (Data Integrity) ตลอดจนรองรับการทำธุรกรรมทางการเงินและการบันทึกประวัติการสั่งซื้ออย่างแม่นยำ
\end{itemize}

\section{กระบวนการพัฒนาโครงการ (Project Development Process)}
โครงการนี้ประยุกต์ใช้วงจรการพัฒนาซอฟต์แวร์แบบอไจล์ (Agile Software Development) ซึ่งเน้นความยืดหยุ่นและการทำงานซ้ำเป็นรอบ (Iterative) เพื่อให้สามารถรับฟังข้อเสนอแนะจากผู้สูงอายุ และนำมาปรับปรุงระบบ (Refactoring) ได้อย่างทันท่วงที โดยมีขั้นตอนหลักดังนี้:

\subsection{ขั้นตอนการดำเนินงาน}
\begin{enumerate}
  \item \textbf{การวิเคราะห์ความต้องการ (Requirement Analysis):} ลงพื้นที่ศึกษาและสัมภาษณ์เชิงลึกกับกลุ่มผู้สูงอายุและผู้ประกอบการร้านค้า เพื่อวิเคราะห์ปัญหา (Pain Points) ในการซื้อขายสินค้าออนไลน์ นำมาสกัดเป็นข้อกำหนดทางวิศวกรรมของซอฟต์แวร์ (Software Requirements Specification)
  \item \textbf{การออกแบบสถาปัตยกรรมระดับระบบ (System Design):} นำข้อกำหนดที่ได้มาสร้างแบบจำลอง เช่น แผนภาพความสัมพันธ์ของเอนทิตี (ER Diagram) แผนภาพกระแสข้อมูล (Data Flow Diagram) รวมถึงการออกแบบหน้าจอผู้ใช้งาน (UI Wireframe) ที่ลดความชับซ้อนให้น้อยที่สุด
  \item \textbf{การพัฒนาและประเมินผล (Implementation and Testing):} เข้าสู่กระบวนการเขียนโค้ด (Coding) เพื่อสร้างระบบจริง พร้อมทำการทดสอบการทำงานควบคู่กันไปในแต่ละสปริ้นท์ (Sprint) เพื่อตรวจสอบข้อบกพร่องและแก้ไขให้ระบบเป็นไปตามมาตรฐานการจัดการซอฟต์แวร์
\end{enumerate}

\section{การออกแบบส่วนติดต่อผู้ใช้งานระดับโครงร่าง (UI/UX Mockups)}
ในขั้นตอนการออกแบบซอฟต์แวร์ คณะผู้จัดทำได้ใช้โปรแกรม Figma ในการสร้างแบบจำลองหน้าจอสมือน (Mockups) เพื่อจำลองประสบการณ์ผู้ใช้งาน (User Experience) และประเมินความเป็นไปได้ทางจิตวิทยาก่อนเริ่มต้นเขียนรหัสโค้ดคำสั่งจริง โดยแต่ละหน้าต่างถูกออกแบบภายใต้แกนความคิดสถาปัตยกรรมเพื่อมวลชน (Universal Design) เพื่อปลดแอกให้ผู้สูงอายุใช้งานได้ด้วยตนเอง มีรายละเอียดวิธีคิดดังนี้:

\section{โครงสร้างสถาปัตยกรรมฐานข้อมูล (Database Structure)}
การออกแบบฐานข้อมูลของแพลตฟอร์ม "Elder Store" ใช้ระบบจัดการฐานข้อมูลเชิงสัมพันธ์ระดับองค์กร (Relational Database) ผ่านเครื่องข่าย PostgreSQL และบริหารจัดการเลเยอร์ข้อมูลผ่าน Prisma ORM โดยโครงสร้างข้อมูลได้รับการจัดรูปแบบเพื่อลดความซ้ำซ้อนทับซ้อน (Database Normalization) ให้อยู่ในระดับ 3NF (Third Normal Form) เพื่อปกป้องความปลอดภัย ยกระดับประสิทธิภาพการคิวรี และรักษาความถูกต้องทางธุรกรรม โดยมีแผนภาพ ER Diagram ดังแสดงในรูปที่ \ref{fig:database-structure}

เพื่อให้ระบบมีเสถียรภาพและพร้อมรองรับการขยายตัวทางทรัพยากรในอนาคต (Scalability) สถาปัตยกรรมข้อมูลทั้งหมดจึงได้ถูกจำแนกกลุ่มความสัมพันธ์ออกเป็น 4 ขั้วหลัก ดังรายละเอียดต่อไปนี้:

\subsection{ระบบการจัดการบัญชีและสิทธิการใช้งาน (User \& Identity Management)}
\begin{itemize}
    \item \textbf{บรรทัดฐานผู้ใช้งาน (User Model):} ตารางแกนหลักสำหรับเก็บข้อมูลรับรองตัวตนบัญชีบุคคลทางไปรษณีย์อิเล็กทรอนิกส์ (Email) คู่กับรหัสผ่านที่เข้ารหัสแล้วแบบฟิสิกส์ (\texttt{passwordHash}) พร้อมประทับตราระดับสิทธิ์เพื่อความปลอดภัยลอจิกด้วยตัวแปร \texttt{role}
    \item \textbf{สมุดบันทึกพิกัด (Address Model):} เชื่อมโยงความสัมพันธ์แบบทางเดียวสู่หลายทาง (1-to-many) กับผู้ใช้งาน เพื่อบันทึกจ่าหน้าซองพิกัดจัดส่งพัสดุ โดยดึงรังวัดละติจูด (\texttt{latitude}) และลองจิจูด (\texttt{longitude}) ไปพักเพื่อเตรียมคำนวณคณิตศาสตร์ระยะทางขนส่ง
    \item \textbf{ระบบความโปร่งใสทางความปลอดภัย (AuditLog \& PasswordResetToken):} มีตารางเก็บบันทึกร่องรอยการเคลื่อนไหวของผู้ดูแลระบบหลังบ้านเพื่อความโปร่งใส (AuditLog) รวมถึงตารางควบคุมคิวโทเค็นอายุสั้นสำหรับการอนุญาตขอกู้คืนรหัสผ่านฉุกเฉิน
\end{itemize}

\subsection{ระบบจัดการแคตตาล็อกและสถาปัตยกรรมภาษาเชิงซ้อน (Product \& Locale Core)}
เพื่อทะลายกำแพงอุปสรรคทางด้านภาษาสำหรับผู้สูงอายุที่มีความหลากหลาย โครงสร้างสินค้าจึงดัดแปลงเทคนิคสถาปัตยกรรมการแปลภาษาดิจิทัล (Localization Model):
\begin{itemize}
    \item \textbf{หมวดหมู่และสินค้า (Category \& Product Models):} ตารางเก็บชุดตัวแปรข้อมูลทางกายภาพที่ตายตัว เช่น ราคา (\texttt{price}), จำนวนของคงคลัง (\texttt{stock}), โควตาโปรโมชันเซล, และลิงก์รูปภาพ
    \item \textbf{ส่วนกระจายภาษา (CategoryLocale \& ProductLocale):} ตารางที่ผูกติดเข้ามาช่วยรับภาระแยกเก็บข้อมูลชื่อสินค้า (\texttt{name}) คำอธิบายจำเพาะ และวัสดุที่เชื่อมโยงด้วยพารามิเตอร์ \texttt{locale} (เช่น 'th' หรือ 'en') ทำให้แพลตฟอร์มสามารถกดสลับภาษาหน้าอินเทอร์เฟซได้ในเสี้ยววินาทีโดยไม่กระทบฐานข้อมูลหลัก
\end{itemize}

\subsection{ระบบรวบรวมตะกร้าและประมวลผลคำสั่งซื้อ (E-Commerce Transactions)}
กลไกการประสานงานเพื่อทำธุรกรรมออกแบบมาให้ปกป้องช่องโหว่การเปลี่ยนแปลงย้อนหลัง (Immutable Logic Constraints):
\begin{itemize}
    \item \textbf{ตะกร้าสินค้า (Cart \& CartItem Model):} เก็บวงจรความสนใจจับจองชั่วคราวของผู้บริโภค หากผู้ใช้อกจากระบบ ตะกร้าของชิ้นนั้นก็ยังบันทึกพ่วงกับไอดีอยู่เสมอ
    \item \textbf{โมดูลคำสั่งซื้อ (Order \& OrderItem Model):} ตารางธุรกรรมที่แข็งแกร่งที่สุด โดยจะปั๊มทำสำเนาข้อมูลราคาของวินาทีนั้น (\texttt{priceAtPurchase}) มาตอกย้ำบันทึกติดประวัติเสมอ เพื่อป้องกันบั๊กฐานข้อมูลพังถล่มในกรณีที่แอดมินหลังบ้านเปลี่ยนราคาสินค้าในอนาคต พร้อมกันนี้ยังบันทึกระยะทางห่างเชิงภูมิศาสตร์ (\texttt{distanceKm}) ยอดค่าส่งบิดเบือนสุทธิ และการหักลบอัตราส่วนลดปะทะกับโครงสร้างคูปองตลาด (Coupon Model) 
\end{itemize}

\subsection{ระบบสนับสนุนเนื้อหาแพลตฟอร์มส่วนหน้า (Content Management System)}
ตารางข้อมูลในคอร์รอง ถูกสร้างเพื่อให้แอดมินบริหารจุดเด่นการตลาดหน้าบ้านได้เบ็ดเสร็จโดยไม่ต้องเขียนโค้ดเพิ่ม ประกอบด้วย:
\begin{itemize}
    \item \textbf{ป้ายโฆษณาโครงข่าย (Banner \& SubBanner Models):} ตารางควบคุมตัวจัดการลิงก์ภาพขยายโฆษณา รวมถึงพิกัดตำแหน่งการสไลด์ในระดับ Layout
    \item \textbf{ระบบศูนย์ช่วยเหลือ (Faq \& FaqLocale Models):} ตารางปัญญาคำถามที่พบบ่อย (Quick Help Responses) ที่เตรียมไว้ชี้แนะข้อสงสัยให้ผู้สูงอายุอัตโนมัติ พร้อมผสานตัวจัดการสลับภาษามาตรฐานระดับโลก
\end{itemize}

\subsection{ความสัมพันธ์ของข้อมูลหลัก (Key Entity Relationships)}
\begin{itemize}
  \item \textbf{Users (ผู้ใช้งาน):} จัดเก็บข้อมูลประจำตัว รหัสผ่านที่เข้ารหัสแล้ว (Hashed Password) และระดับสิทธิ์การอนุญาต (Role-based Access Control) แยกแยะระหว่างบัญชีผู้ซื้อ (ผู้สูงอายุ) และผู้ขาย (ร้านค้า)
  \item \textbf{Products (สินค้า):} ประกอบด้วยข้อมูลพรรณนาสินค้า หมวดหมู่ ราคา จำนวนสินค้าคงคลัง และที่สำคัญคือฟิลด์สำหรับจัดการ "คำอธิบายรูปภาพทางเลือก (Alt text)" เพื่อให้โปรแกรมอ่านหน้าจอ (Screen Reader) สามารถทำงานตามมาตรฐาน WCAG ได้อย่างสมบูรณ์
  \item \textbf{Orders (คำสั่งซื้อ):} โครงสร้างข้อมูลที่เชื่อมโยงตารางผู้ใช้งานและตารางสินค้าเข้าด้วยกัน ใช้จัดเก็บสถานะสถานะการสั่งซื้อ (เช่น รอชำระเงิน, กำลังจัดส่ง, สำเร็จแล้ว) และข้อมูลที่อยู่สำหรับการจัดส่ง
  \item \textbf{Payments (การชำระเงิน):} ตารางเก็บบันทึกหลักฐานการทำธุรกรรม (Transaction Log) ภาพหลักฐานการโอนเงิน (สลิป) และตรรกะการเชื่อมต่อตรวจสอบยอดการชำระผ่านระบบ QR Code PromptPay
  \item \textbf{Analytics (การวิเคราะห์สถิติ):} บันทึกร่องรอยการเข้าชมระบบ การคลิกสินค้า และยอดขายรวม เพื่อเป็นข้อมูลนำเข้าสำหรับระบบแนะนำสินค้า (Product Recommendation) และ แดชบอร์ดสรุปยอดของฝั่งร้านค้า
\end{itemize}

\section{วิธีการทดสอบระบบ (System Testing Methodology)}
การทดสอบระบบเป็นหนึ่งในพันธกิจอับดับแรกที่จะรับประกันคุณภาพซอฟต์แวร์ (Software Quality Assurance) เพื่อยืนยันว่าทุกฟังก์ชันมีความปลอดภัย แข็งแกร่ง (Robustness) และเป็นมิตรต่อผู้ใช้งานกลุ่มผู้สูงอายุ โดยโครงการนี้ได้กำหนดกรอบการทดสอบแบ่งออกเป็น 4 ระดับ ดังนี้:

\begin{itemize}
  \item \textbf{การทดสอบระดับหน่วย (Unit Testing):}  
  ตรวจสอบความถูกต้องของฟังก์ชันแต่ละส่วนที่อยู่ในระดับเล็กที่สุด เช่น อัลกอริทึมในการคำนวณราคาสินค้ารวมในตะกร้า การกรองประเภทสินค้า และการยืนยันรหัสผ่าน เพื่อรับประกันว่าไม่มีข้อผิดพลาดเชิงตรรกะ (Logical Errors)

  \item \textbf{การทดสอบการทำงานข้ามระบบ (Integration Testing):}  
  ประเมินความสมบูรณ์ในการสื่อสารผ่าน API ระหว่าง Frontend, Backend และฐานข้อมูล เช่น ทดสอบสถานการณ์ที่ผู้ใช้กด "สั่งซื้อสินค้า" ไปจนถึงการ "ตัดจำนวนสินค้าคงคลังในฐานข้อมูล" เพื่อป้องกันปัญหาข้อมูลสูญหายหรือไม่ตรงกัน

  \item \textbf{การทดสอบการยอมรับจากผู้ใช้งานจริง (User Acceptance Testing - UAT):}  
  นำซอฟต์แวร์ต้นแบบ (Prototype) ไปทดสอบในสภาวะแวดล้อมจริง โดยเชิญกลุ่มเป้าหมายซึ่งประกอบไปด้วย "กลุ่มผู้สูงอายุ" และ "ผู้ประกอบการร้านค้า" มาลองใช้งานจริง โดยแพลตฟอร์มนี้จะเน้นจำลองการทดสอบบนอุปกรณ์สมาร์ตโฟนเป็นหลัก เพื่อประเมินผลค่าความพึงพอใจ ศึกษาความเข้าใจของการนำทางเว็บ (Navigation) และความยากง่ายในการทำธุรกรรม

  \item \textbf{การทดสอบมาตรฐานการเข้าถึง (Accessibility Testing):}  
  ระบบ Elder Store ได้ทดสอบเทียบเคียงกับมาตรฐานข้อกำหนดสากล WCAG 2.1 (Web Content Accessibility Guidelines) เพื่อสร้างความมั่นใจแก่ผู้มีข้อจำกัดทางสรีระ โดยเน้นย้ำทดสอบหัวข้อต่างๆ ดังนี้:
  \begin{itemize}
      \item \textbf{อัตราส่วนคอนทราสต์ (Contrast Ratio):} วัดความต่างของสีพื้นหลังและสีข้อความให้เป็นไปตามเกณฑ์ระดับ AA/AAA เพื่อให้ผู้สูงอายุสายตาฝ้าฟางอ่านได้ชัดเจน
      \item \textbf{พื้นที่การสัมผัส (Touch Targets):} วัดและปรับขนาดปุ่มกดให้กว้างเพียงพอ เพื่อลดโอกาสการกดผิดจากปัญหาการควบคุมกล้ามเนื้อมือผู้สูงอายุ
      \item \textbf{การเข้าถึงด้วยปุ่มกด (Keyboard Usability):} ตรวจสอบว่าทุกองค์ประกอบบนเว็บไซต์สามารถเข้าถึงได้โดยไม่ต้องใช้เมาส์
      \item \textbf{ระบบสนับสนุนตัวช่วยอ่านหน้าจอ (Screen Reader Compatibility):} ทดสอบความรองรับกับซอฟต์แวร์ช่วยอ่านเพื่อผู้พิการทางสายตา
  \end{itemize}
\end{itemize}

\section{สรุป}
แผนการดำเนินงานและโครงสร้างของแพลตฟอร์มทั้งหมดที่กล่าวมา สะท้อนให้เห็นถึงความมุ่งมั่นของโครงการ Elder Store ในการพัฒนาเทคโนโลยีที่สามารถนำไปใช้งานได้จริงอย่างปลอดภัย \cite{ecommerce_reference_here} การออกแบบเชิงบูรณาการตั้งแต่งานโครงร่าง Figma ไปจนถึงเซิร์ฟเวอร์ฐานข้อมูล ควบคู่กับการยึดหลักเกณฑ์ความครอบคลุมสากล (WCAG 2.1) ไม่เพียงแต่ช่วยทลายขีดจำกัดทางดิจิทัลของผู้สูงอายุเท่านั้น แต่ยังส่งเสริมให้ผู้ประกอบการร้านค้าสามารถต่อยอดและให้บริการแก่กลุ่มผู้สูงอายุได้อย่างไร้รอยต่อ
