\chapter{\ifenglish Background Knowledge and Theory\else ทฤษฎีที่เกี่ยวข้อง\fi}

การทำโครงการ Elder Store มีเป้าหมายเพื่อสร้างแพลตฟอร์มการพาณิชย์อิเล็กทรอนิกส์ (E-Commerce) ที่ช่วยอำนวยความสะดวกให้ผู้สูงอายุสามารถเข้าถึงสินค้าและบริการได้อย่างไร้รอยต่อ โดยเน้นการออกแบบที่เรียบง่าย ปลอดภัย และเหมาะสมกับข้อจำกัดทางกายภาพและการรับรู้ของผู้สูงอายุ เพื่อให้บรรลุวัตถุประสงค์ดังกล่าว คณะผู้จัดทำได้ศึกษาและรวบรวมทฤษฎี หลักการทางวิศวกรรมซอฟต์แวร์ \cite{pressman2014} หลักการออกแบบ และเทคโนโลยีที่เกี่ยวข้อง ซึ่งประกอบด้วยหัวข้อดังต่อไปนี้

\section{ทฤษฎีปัจจัยทางกายภาพและปริชานของผู้สูงอายุ (Physiological and Cognitive Factors)}
การเข้าสู่สังคมผู้สูงอายุ (Aging Society) ทำให้กลุ่มผู้สูงอายุมีบทบาทสำคัญในตลาดออนไลน์มากขึ้น อย่างไรก็ตาม ผู้สูงอายุมีข้อจำกัดทางสรีรวิทยาที่ส่งผลต่อการใช้งานเทคโนโลยี \cite{accessibility_elderly_2012} ซึ่งระบบที่ดีต้องคำนึงถึงปัจจัยต่อไปนี้:
\begin{itemize}
    \item \textbf{การมองเห็น (Visual Impairment):} กลุ่มผู้สูงอายุมักประสบภาวะสายตายาวหรือสายตาพร่ามัว ทำให้ยากต่อการอ่านตัวอักษรขนาดเล็ก หรือการแยกแยะองค์ประกอบที่มีความเปรียบต่างของสีต่ำ (Low Contrast) ซึ่งเป็นอุปสรรคหลักในการกวาดสายตาอ่านหน้าจอ \cite{wai_older_users}
    \item \textbf{การควบคุมกล้ามเนื้อมัดเล็ก (Motor Skill Decline):} การควบคุมนิ้วมือเพื่อสัมผัสหน้าจอสมาร์ตโฟนอาจมีความแม่นยำลดลง ทำให้การกดปุ่มที่มีขนาดเล็ก หรือปุ่มที่เรียงชิดกันเกินไปทำได้ยาก และเกิดความผิดพลาดจากการสัมผัสคลาดเคลื่อนบ่อยครั้ง
    \item \textbf{ระบบความจำและการเรียนรู้ (Cognitive Load and Memory):} ความสามารถในการจดจำระยะสั้นที่ถดถอย ทำให้ผู้สูงอายุอาจเกิดความสับสนเมื่อต้องเผชิญกับระบบที่มีขั้นตอนการทำงานซับซ้อน หรือมีหน้าจอที่อัดแน่นไปด้วยข้อมูลจำนวนมากเกินไป (Information Overload) นำไปสู่อัตราการละทิ้งตะกร้าสินค้า
\end{itemize}
ข้อจำกัดเหล่านี้เป็นปัจจัยพื้นฐานสำคัญที่โครงการ Elder Store นำมาเป็นกรอบแนวคิดในการออกแบบระบบ เพื่อให้ได้แพลตฟอร์มที่ตอบโจทย์และแก้ปัญหา (Pain Point) ได้อย่างแท้จริง

\section{หลักการออกแบบปฏิสัมพันธ์และประสบการณ์ผู้ใช้สำหรับผู้สูงอายุ (Elderly-Friendly UI/UX)}
เพื่อให้แพลตฟอร์มเป็นมิตรต่อผู้สูงอายุ โครงการได้นำหลักการออกแบบเพื่อคนทั้งมวล (Inclusive Design) \cite{inclusive2022} และมาตรฐานการเข้าถึงโครงสร้างข้อมูลเว็บ (Web Content Accessibility Guidelines: WCAG) \cite{wcag2023, wcag22_2023} มาประยุกต์ใช้ ดังนี้:
\begin{itemize}
    \item \textbf{ความชัดเจนของข้อความ (Typography and Readability):} เลือกใช้ฟอนต์ที่อ่านง่าย สบายตา ไม่มีหัว (Sans-serif) หรือฟอนต์มาตรฐานที่มีความหนาพอสมควร โดยกำหนดขนาดตัวอักษรขั้นต่ำให้ใหญ่กว่ามาตรฐานเว็บไซต์ทั่วไป เพื่อสอดรับกับหลักภาระการรับรู้ \cite{nielsen2000}
    \item \textbf{ความเปรียบต่างของสี (Color Contrast):} ใช้สีพื้นหลังและสีตัวอักษรที่ตัดกันอย่างชัดเจนตามอัตราส่วนที่ WCAG แนะนำ \cite{color_contrast_wcag} ลดการใช้สีสันที่ฉูดฉาดเกินไปเพื่อป้องกันความเมื่อยล้าของดวงตา
    \item \textbf{พื้นที่การจัดวางและการสัมผัส (Spacing and Touch Targets):} ขยายขนาดเป้าหมายการกด (Touch Targets) ให้เอื้อต่อการสัมผัส (ควรมีขนาดอย่างน้อย 44x44 พิกเซล) \cite{mobileux2021} และเว้นระยะห่างของปุ่มเพื่อลดกระบวนการที่ผิดพลาด
    \item \textbf{การนำทางที่เข้าใจง่าย (Simplified Navigation):} โครงสร้างหน้าเว็บจะถูกออกแบบในลักษณะเส้นตรง (Linear Flow) ขจัดเมนูย่อยที่ซับซ้อนออก เพื่อสะท้อนหลักการทฤษฎี “อย่าทำให้ผู้ใช้ต้องคิดเผื่อ” \cite{krug2000} และแสดงขั้นตอนการประมวลผลเป็นลำดับ
\end{itemize}

\section{การวิเคราะห์ระบบที่มีใช้อยู่ในปัจจุบัน (Analysis of Existing Systems)}
ปัจจุบันมีแพลตฟอร์มพาณิชย์อิเล็กทรอนิกส์ในตลาดภูมิภาคมากมาย \cite{ecommerce2019} แต่ส่วนใหญ่ถูกออกแบบมาเพื่อรองรับกลุ่มเป้าหมายในวงกว้าง (Mass Market) ทำให้มีคุณลักษณะก่ออุปสรรคต่อผู้สูงอายุ
\begin{itemize}
    \item \textbf{แพลตฟอร์มตลาดกลาง (Shopee / Lazada):} 
    \begin{itemize}
        \item \textit{ข้อดี:} มีระบบซื้อขายครบวงจร มีสินค้าให้เลือกหลากหลาย และมีระบบโลจิสติกส์ที่ได้มาตรฐาน
        \item \textit{ข้อจำกัดสำหรับผู้สูงอายุ:} หน้าจอหลักของแอปพลิเคชันมีการนำเสนอโปรโมชัน แบนเนอร์โฆษณาเคลื่อนไหว และหน้าต่างแจ้งเตือน (Pop-up) จำนวนมาก ซึ่งสร้างความรำคาญใจและเบี่ยงเบนความสนใจ ขั้นตอนการพยายามเก็บคูปองมีความสลับซับซ้อน
    \end{itemize}
    \item \textbf{โซเชียลคอมเมิร์ซทางการค้า (Facebook / LINE):}
    \begin{itemize}
        \item \textit{ข้อดี:} เป็นแอปพลิเคชันที่ผู้สูงอายุใช้งานเพื่อการสื่อสารอยู่แล้วในชีวิตประจำวัน จึงมีความคุ้นชินสูง
        \item \textit{ข้อจำกัดสำหรับผู้สูงอายุ:} ขาดระบบตะกร้าสินค้าที่เป็นมาตรฐาน และมีความเสี่ยงสูงมากต่อการพบเจอมิจฉาชีพ (Scams) ที่หลอกลวงขายสินค้า
    \end{itemize}
\end{itemize}
\textbf{ข้อแตกต่างของ Elder Store:} ระบบจะส่งมอบประสบการณ์ที่ขจัดสิ่งรบกวน (Distractions-free) \cite{uxdesign2023} อย่างแบนเนอร์หรือสคริปต์ที่ไม่จำเป็นทิ้งทั้งหมด นำเสนอเพียงฟังก์ชันหลักคือ ค้นหาสินค้าจากภาพขนาดใหญ่ และปุ่มคำสั่งซื้อที่เด่นชัด เพื่อมุ่งเป้าสู่การทำธุรกรรมแบบมั่นคง 

\section{ความปลอดภัยและความน่าเชื่อถือในการทำธุรกรรมออนไลน์ (Trust in E-Commerce)}
อุปสรรคที่ใหญ่ที่สุดอย่างหนึ่งของผู้สูงอายุคือความกลัวทำพาสเวิร์ดหายและการทำข้อมูลรั่วไหล โครงการจึงส่งเสริมทฤษฎีความเชื่อมั่น (Theory of Trust) \cite{shneiderman2010} มาประยุกต์ใช้ในระบบดังนี้:
\begin{itemize}
    \item \textbf{กระบวนการเข้าสู่ระบบที่ปลอดภัยระดับสูง:} มีระบบยืนยันตัวตนที่ไม่ต้องพึ่งพาการจดจำรหัสผ่านยาวๆ ลดความเครียดทางเทคโนโลยี
    \item \textbf{ระบบโครงสร้างการชำระเงินแห่งชาติ (National E-Payment):} สนับสนุนรูปแบบการชำระเงินโดยแนบสลิปผ่านทางระบบพร้อมเพย์ (PromptPay) \cite{promptpay2021} ซึ่งหลีกเลี่ยงการให้ผู้สูงอายุต้องพิมพ์รหัสบัตรเครดิตลงอินเทอร์เน็ตโดยตรง สร้างความสบายใจได้ดีที่สุด
    \item \textbf{ความโปร่งใสในข้อมูล (Information Transparency):} แสดงรายละเอียดสินค้า สรรพคุณ ราคา และวิธีการจัดส่งที่ชัดเจน ไม่หมกเม็ด
    \item \textbf{ช่องทางการช่วยเหลือ (Support Center):} มีปุ่มช่วยเหลือสำหรับติดต่อผู้ดูแลระบบเมื่อต้องการคำแนะนำ
\end{itemize}

\section{เทคโนโลยีเชิงสถาปัตยกรรม (Platform Technologies)}
ในการพัฒนาแพลตฟอร์ม ผู้จัดทำเลือกใช้เทคโนโลยีเว็บคอร์สมัยใหม่ (Modern Web Stack) เพื่อความเสถียรและสามารถขยายขนาดได้จริง:
\begin{itemize}
    \item \textbf{การออกแบบเว็บไซต์แบบตอบสนอง (Responsive Web Design):} โครงสร้างหน้าจอประยุกต์ทฤษฎีการตอบสนองเรขาคณิต \cite{responsiveweb} ร่วมกับเฟรมเวิร์ก Tailwind CSS \cite{tailwind2023} เพื่อให้หน้าเว็บยืดหยุ่นไร้ขีดจำกัด (Cross-platform) บนทุกอุปกรณ์
    \item \textbf{สถาปัตยกรรมเว็บเบื้องหน้าและเบื้องหลัง (Frontend \& Backend Architecture):} มุ่งเน้นการใช้ไลบรารี React \cite{react2023} ผสานกับเฟรมเวิร์ก Next.js \cite{nextjs2023} เพื่อให้เกิดการสะท้อนกลับ (Feedback) แก่ผู้ใช้งานทันที การจัดการฐานข้อมูลดำเนินผ่านตารางข้อมูลเชิงสัมพันธ์ PostgreSQL \cite{postgresql2023} ส่งผลให้การสืบค้นประวัติออเดอร์ของผู้สูงอายุและระบบหลังบ้าน แข็งแรงตามหลักวิศวกรรม \cite{fielding2000}
\end{itemize}
