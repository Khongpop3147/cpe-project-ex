\section{\ifenglish Project rationale\else ที่มาและความสำคัญของโครงงาน\fi}
ปัจจุบัน โลกกำลังก้าวเข้าสู่ยุค "สังคมผู้สูงอายุอย่างสมบูรณ์" (Super-Aged Society) ซึ่งส่งผลให้โครงสร้างประชากรและรูปแบบการใช้ชีวิตเกิดการเปลี่ยนแปลงอย่างมีนัยสำคัญ \cite{accessibility_elderly_2012} ประกอบกับการเติบโตอย่างก้าวกระโดดของเทคโนโลยีและแพลตฟอร์มพาณิชย์อิเล็กทรอนิกส์ (E-Commerce) ที่เข้ามามีบทบาทขั้นพื้นฐานในการอำนวยความสะดวกในชีวิตประจำวัน แม้ว่าแพลตฟอร์มชอปปิงออนไลน์ที่มีอยู่ในปัจจุบันจะมีความหลากหลายและบริการที่ครอบคลุม แต่ส่วนใหญ่ถูกออกแบบและพัฒนาขึ้นมาเพื่อสอดรับกับพฤติกรรมของกลุ่มคนรุ่นใหม่และวัยทำงาน (Digital Natives) เป็นหลัก \cite{ecommerce2019}

จากช่องว่างดังกล่าว ส่งผลให้กลุ่มผู้สูงอายุมักเผชิญกับอุปสรรคและความยากลำบากในการเข้าถึงเทคโนโลยีเหล่านี้ ไม่ว่าจะเป็นปัญหาด้านข้อจำกัดทางสรีรวิทยา เช่น การมองเห็นตัวอักษรหรือไอคอนที่มีขนาดเล็กเกินไป ความซับซ้อนของส่วนติดต่อผู้ใช้งาน (User Interface) ที่เต็มไปด้วยข้อความโฆษณาแฝงและเมนูย่อยที่ก่อให้เกิดความสับสนและเป็นภาระต่อระบบความจำ (Cognitive Overload) \cite{wai_older_users} ตลอดจนความกังวลด้านความปลอดภัยในการทำธุรกรรมออนไลน์ที่เสี่ยงต่อการถูกมิจฉาชีพหลอกลวง (Scams)

ด้วยเหตุนี้ คณะผู้จัดทำจึงได้เล็งเห็นถึงความจำเป็นในการพัฒนาแพลตฟอร์ม "Elder Store" ซึ่งเป็นระบบร้านค้าออนไลน์ที่ออกแบบมาโดยมีผู้สูงอายุเป็นศูนย์กลาง (Elderly-Centric Design) โครงการนี้มุ่งเน้นการขจัดความเหลื่อมล้ำทางดิจิทัล (Digital Divide) โดยประยุกต์ใช้หลักการออกแบบเพื่อมวลชน (Inclusive Design) \cite{inclusive2022} เพื่อลดทอนความซับซ้อน สะท้อนหลักการปฏิสัมพันธ์ของมนุษย์กับคอมพิวเตอร์แบบไร้ข้อจำกัด \cite{shneiderman2010} นำเสนอข้อมูลที่อ่านง่าย ชัดเจน และมีระบบการสั่งซื้อที่ปลอดภัย ทั้งนี้เพื่อยกระดับคุณภาพชีวิตและเพิ่มสิทธิในการเข้าถึงสินค้าและบริการที่จำเป็นของผู้สูงอายุ ให้สามารถใช้ชีวิตในยุคดิจิทัลได้อย่างเป็นอิสระและพึ่งตนเองได้

\section{\ifenglish Objectives\else วัตถุประสงค์ของโครงงาน\fi}
\begin{enumerate}
    \item เพื่อศึกษาและวิเคราะห์ข้อจำกัด พฤติกรรม และความต้องการในการใช้งานแพลตฟอร์มพาณิชย์อิเล็กทรอนิกส์ของกลุ่มประชากรผู้สูงอายุ
    \item เพื่อออกแบบและพัฒนาแพลตฟอร์มร้านค้าออนไลน์ "Elder Store" ที่มีประสบการณ์ผู้ใช้งานและสถาปัตยกรรม (UX/UI) ตอบโจทย์กับข้อจำกัดทางสรีรวิทยาของผู้สูงอายุและสั่งซื้อสินค้าได้ง่าย
    \item เพื่อพัฒนาระบบกรองข้อมูลและแนะนำสินค้าที่มีความเสถียร ช่วยลดตรรกะความซับซ้อน (Reduce Complexity) และประหยัดเวลาในการค้นหา
    \item เพื่อส่งเสริมคุณภาพชีวิตของผู้สูงอายุผ่านการประยุกต์ใช้เทคโนโลยีสมัยใหม่ และอ้างอิงการประเมินความพึงพอใจการใช้งานระบบในสภาวะปัจจัยควบคุมทางธุรกิจจริง
\end{enumerate}

\section{\ifenglish Project scope\else ขอบเขตของโครงงาน\fi}

\subsection{\ifenglish Hardware scope\else ขอบเขตด้านฮาร์ดแวร์\fi}
\begin{itemize}
    \item \textbf{หน่วยผู้ใช้งาน (Client-side):} ผู้ใช้งานสามารถเข้าถึงระบบผ่านอุปกรณ์อิเล็กทรอนิกส์พื้นฐานที่รองรับการเชื่อมต่อเครือข่ายอินเทอร์เน็ตและมีเวเบราว์เซอร์มาตรฐาน เช่น สมาร์ตโฟน แท็บเล็ต และคอมพิวเตอร์ โดยไม่จำเป็นต้องติดตั้งอุปกรณ์เสริมทับ
    \item \textbf{หน่วยประมวลผลศูนย์กลาง (Server-side):} การใช้งานเครื่องคอมพิวเตอร์แม่ข่าย (Cloud Server) เพื่อรองรับการประมวลผลตารางฐานข้อมูลและให้บริการเว็บไซต์ที่มีความพร้อมใช้งานตลอด 24 ชั่วโมง
\end{itemize}

\subsection{\ifenglish Software scope\else ขอบเขตด้านซอฟต์แวร์\fi}
ระบบจะครอบคลุมโมดูลการทำงานหลัก (Core Features) ดังต่อไปนี้:
\begin{itemize}
    \item \textbf{ระบบสมาชิก:} การลงทะเบียนและการเข้าสู่ระบบด้วยกระบวนการยืนยันตัวตนที่ลดภาระความจำทางเทคโนโลยี
    \item \textbf{ระบบแคตตาล็อกนำเสนอสินค้า (Product Catalog):} การแสดงผลรายการสินค้าที่เน้นรูปภาพขนาดใหญ่และตัวอักษรที่หนักแน่น พร้อมฟังก์ชันแนะนำสินค้าอัตโนมัติ
    \item \textbf{ระบบสั่งซื้อและการชำระเงิน (Checkout Flow):} ตะกร้าสินค้าและการชำระเงินที่ถูกรวบรัดออกแบบให้เป็นอัลกอริทึมเส้นตรง (Linear Step Operation) ตัดทิ้งเงื่อนไขเมนูที่ชวนหลงทาง
    \item \textbf{ระบบบริหารข้อมูลหลังบ้าน (Admin Dashboard):} กระดานสำหรับผู้จัดการแพลตฟอร์มในการควบคุมสินทรัพย์ดิจิทัล ตรวจสอบคำสั่งซื้อ อนุมัติสลิปโอนเงิน และดูรายงานภาพรวมได้อย่างครอบคลุม
\end{itemize}

\subsection{\ifenglish Demographic scope\else ขอบเขตด้านกลุ่มเป้าหมาย\fi}
กลุ่มประชากรเป้าหมายหลักในการใชังานและประเมินผลประสิทธิภาพระบบระดับผู้ปฏิบัติงาน คือ ผู้สูงอายุที่มีอายุตั้งแต่ 60 ปีขึ้นไปที่มีทักษะการใช้งานอุปกรณ์สัมผัสขั้นพื้นฐาน และเครือข่ายกลุ่มพาร์ทเนอร์/ผู้ประกอบการที่ต้องการใช้แพลตฟอร์มช่วยกระจายสินค้าที่สอดคล้องกับพฤติกรรมผู้สูงวัย

\section{\ifenglish Expected outcomes\else ประโยชน์ที่คาดว่าจะได้รับ\fi}
\begin{itemize}
    \item \textbf{ด้านจิตวิทยาและสรีรวิทยาผู้สูงอายุ:} ช่วยให้ผู้สูงอายุได้รับแพลตฟอร์มการจับจ่ายที่มั่นคงแข็งแรงต่อสายตาและนิ้วมือ ลดการพึ่งพาผู้อื่น และสร้างความภาคภูมิใจในการควบคุมคอมพิวเตอร์อย่างลื่นไหล
    \item \textbf{ด้านวิศวกรรมข้อมูลธุรกิจ (Business Engineering):} สร้างระบบสถาปัตยกรรมธุรกิจใหม่ ช่วยให้ผู้ประกอบการเจาะตลาดเทรนด์เศรษฐกิจผู้สูงวัย (Silver Economy) พร้อมการคัดกรองข้อมูลธุรกรรมที่รวดเร็ว
    \item \textbf{ด้านประสบการณ์สถาปัตยกรรมระบบ (System Architecture Design):} ได้รับระบบต้นแบบ (Prototype) ที่พิสูจน์แล้วว่าการตั้งค่าอินเทอร์เฟซแบบลดสิ่งเร้าตา (Distraction-free) สามารถเพิ่มโอกาสความสำเร็จในการปิดยอดขายได้จริง \cite{uxtrends2023}
    \item \textbf{ด้านสังคมวิทยา:} เป็นหนึ่งในกลไกซอฟต์แวร์นวัตกรรมที่สนับสนุนโครงสร้างพื้นฐานเตรียมพร้อมรับมือปรากฏการณ์สังคมผู้สูงอายุ (Aging Society Infrastructure) ของประเทศ
\end{itemize}

\section{\ifenglish Technology and tools\else เทคโนโลยีและเครื่องมือที่ใช้ในการพัฒนา\fi}

\subsection{\ifenglish Hardware technology\else เทคโนโลยีด้านฮาร์ดแวร์\fi}
\begin{itemize}
    \item เครื่องคอมพิวเตอร์สำหรับนักพัฒนา (Development Workstations) รองรับระบบปฏิบัติการมาตรฐานสำหรับการเขียนภาษาโปรแกรมมิ่งและจำลองโลคอลโฮสต์คอมไพล์เลอร์ (Localhost Compiler)
    \item สมาร์ตโฟนและเครื่องแท็บเล็ตหน้าจอต่างๆ สำหรับการทดสอบออโต้เรนเดอร์แบบตอบสนองบนจอประมวลผลจริง (Responsive Device Validating)
\end{itemize}

\subsection{\ifenglish Software technology\else เทคโนโลยีด้านซอฟต์แวร์\fi}
ซอฟต์แวร์และเครื่องมือที่ใช้ประกอบการวิจัยพัฒนาโครงการนี้ขับเคลื่อนด้วยโครงสร้างเทคโนโลยีร่วมสมัย (Modern Tech Stack) ดังนี้:
\begin{itemize}
    \item \textbf{Frontend (ส่วนติดต่อผู้ใช้งานไคลเอนต์):} นิยามตรรกะปุ่มกดด้วยไลบรารี \textbf{React.js} \cite{react2023} ร่วมกับเฟรมเวิร์กจัดการสไตล์ \textbf{Tailwind CSS} \cite{tailwind2023} ทำให้กระบวนการวางโครงข่ายร่างผลลัพธ์ (Layout) ตลอดจนการฝังค่าความเปรียบต่างฟอนต์สีสำหรับผู้สูงอายุ ทำงานลุล่วงอย่างเป็นระบบ
    \item \textbf{Backend (โครงสร้างเซิร์ฟเวอร์):} ประยุกต์ใช้แพลตฟอร์ม \textbf{Next.js} \cite{nextjs2023} ควบคู่กับภาษา TypeScript ซึ่งโดดเด่นในกระบวนการพรีเรนเดอร์เนื้อหาฝั่งเซิร์ฟเวอร์ (Server-Side Rendering) เพิ่มประสิทธิภาพความเร็วลดปิงระหว่างการวาดข้อมูลหน้าเว็บอย่างมหาศาล
    \item \textbf{Database (ฐานข้อมูลเชิงสัมพันธ์):} เลือกระบบจัดการคลังข้อมูลเอนติทีระดับสูง \textbf{PostgreSQL} \cite{postgresql2023} เข้าสูตรสำเร็จตามทฤษฎีจัดการสถาปัตยกรรมอาร์โอเอส \cite{fielding2000} มาบริหารตัวแปรผู้ใช้ และสปีดประมวลยอดบิลสินค้า
    \item \textbf{Authentication (กลไกสิทธิ์การเข้าถึง):} ควบคุมดูแลความปลอดภัยเชิงตัวแปรเซสชันด้วยแนวทางโมเดล (Entity Security) บนเซกเมนต์ป้องกันการดักจับข้อมูลคุกกี้ ลดวิกฤตคุกคามทางไซเบอร์ตามจรรยาบรรณวิศวกรรมซอฟต์แวร์ \cite{pressman2014}
    \item \textbf{Accessibility (มาตรฐานข้อบังคับระดับโลก):} ทำการวัดระยะอัตราส่วนและออกแบบโครงสร้างแอปพลิเคชันทุกเมทริกซ์ให้คล้องตามประกาศมาตรฐาน \textbf{WCAG 2.1} \cite{wcag2023} รองรับโปรแกรมพากย์อ่านเนื้อหา (Screen Reader) ปกป้องสิทธิผู้มีปัญหาทางกายภาพให้สมบูรณ์แบบ
\end{itemize}

เพื่อให้กระบวนการพัฒนาโครงงานบรรลุวัตถุประสงค์โดยไม่ขັດต่อหลักเส้นเวลาการทำงานแบบว่องไว (Agile) \cite{beck2001} คณะผู้จัดทำได้วางแผนกระจายความรับผิดชอบเชิงระบบอุตสาหกรรมใน 4 แท่งสำคัญ ดังนี้:
\begin{itemize}
    \item \textbf{ฝ่ายวิเคราะห์แพตเทิร์นกระแสความต้องการผู้ใช้ (Requirement Analysis):} รับผิดชอบลงเครื่องมือสนาม สัมภาษณ์วัดสเกลปัญหา (Pain points) กลุ่มประชากรรุ่นอาวุโส นำผลที่ได้มาตีรหัสร่างสูตรแปลงเป็นข้อกำหนดของระบบ (System Requirements) ให้สมบูรณ์แบบ \cite{pressman2014}
    \item \textbf{ฝ่ายสกัดกรองดีไซน์และสถาปัตยกรรมปฏิสัมพันธ์ (UI/UX Architect):} มีหน้าที่จับหลักการอารยสถาปัตย์ (Inclusive Design) มากางเป็นเส้นทางบรรทัดฐานผู้ใช้ (User Flow) จนกลั่นออกมาเป็นตัว Mock-up อินเทอร์เฟซที่พาสซิ่งผ่านเกณฑ์ WCAG
    \item \textbf{ฝ่ายขุดเจาะเขียนคำสั่งโปรโตคอล (System Development \& Testing):} รับเหมาหน้างานในการดีคอดิ้งสร้างเซิร์ฟเวอร์คอมโพเนนต์ทั้ง 2 แกน (Frontend และ Backend) วางระเบียบพอร์ท API ทำการคอมไพล์เจาะบั๊กให้รวบรัดที่สุด เพื่อรีลีซเวอร์ชันที่แข็งแรงที่สุด
    \item \textbf{ฝ่ายจัดการพอร์ตโฟลิโอเอกสารทางวิชาการ (Documentation Control):} มีภาระหน้าตารักษาบันทึกล็อกไฟล์ รวบรวมข้อมูลดีไซน์หลักการสร้างอัลกอริทึมเพื่อนำมาเรียบเรียงเข้าเล่มปริญญานิพนธ์ฉบับทางการ และปั้นรายงานสถิติประกอบการนำเสนอ
\end{itemize}

\section{\ifenglish Team Roles\else บทบาทของสมาชิกในทีม\fi}
ทีมคณะผู้ศึกษาวิจัย Elder Store จัดสรรพละกำลังออกเป็น 3 ยูนิตเชิงเทคนิคเฉพาะทาง ดังนี้:
\begin{itemize}
    \item \textbf{ผู้ควบรวมคุมสถาปัตยกรรมหลังบ้าน (Backend Server Architect):} ร่างวงจรคอนฟิกูเรชันเซิร์ฟเวอร์ ออกแบบโครงสร้าง Schema โมเดลตารางผ่าน PostgreSQL พร้อมคุมเข้มสร้างเส้นทางพอร์ต API เพื่อขนส่ง Data โหลดระหว่างคลัสเตอร์ ป้องกันการโจมตีจากช่องโหว่การไม่เข้ารหัสในส่วนของเซสชันผู้สูงอายุ
    \item \textbf{ผู้พัฒนาอินเทอร์เฟซส่วนสั่งการคอมพิวเตอร์ (Frontend Developer - Web):} ตำแหน่งดูแลกลไก React.js ประกอบร่างกับชิ้นส่วนหน้าต่าง Tailwind เพื่อการันตีว่าแพลตฟอร์มไซตติ้งบนคอมพิวเตอร์จะทำงานด้วยความเข้าใจง่าย มีองค์ประกอบเด่นชัดตามลำดับชั้นวิชวล (Clear Visual Hierarchy) ไม่รกตาเหมือนแคร็กเว็บเกรดต่ำ
    \item \textbf{ผู้พัฒนาอินเทอร์เฟซส่วนสั่งการฉบับพกพา (Frontend Developer - Mobile):} รับหน้าที่แปรสภาพโค้ดออโต้เรนเดอร์ลงให้จอขนาดเล็ก (Responsive UI) เช่น สมาร์ตโฟน ซึ่งเป็นฮาร์ดแวร์พื้นฐานที่คนแก่ถือครองเป็นหลัก ผู้รับผิดชอบจะต้องตีโจทย์รันโมดูลควบคุมองศาการขยายปลายนิ้ว (Touch Targets Area) เพื่อแก้เงื่อนงำปัญหาการกดลั่นของนิ้วมือใหญ่ๆ ให้เด็ดขาด
\end{itemize}

\section{\ifenglish%
Impacts of this project on society, health, safety, legal, and cultural issues
\else%
ผลกระทบด้านสังคม สุขภาพ ความปลอดภัย กฎหมาย และวัฒนธรรม
\fi}

โครงการ Elder Store ไม่ได้จบลงเพียงแค่สคริปต์เว็บไซต์อีคอมเมิร์ซธรรมดา แต่ถือเป็นสะพานสร้างกระบวนทัศน์ที่จะมีผลกระทบต่อภาพรวมของชาติใน 5 โทนความหมาย ดังนี้:

\textbf{1. ผลกระทบด้านสังคม (Social Impacts):} 
เป็นส่วนร่วมบรรเทาปัญหาความเหลื่อมล้ำทางดิจิทัลแบบสมบูรณ์ (Digital Divide Eradication) สร้างพื้นที่ปลอดภัยมอบสิทธิให้ผู้เฒ่าผู้แก่สามารถสั่งซื้อปัจจัยดำรงชีพบนกระดานอินเทอร์เน็ตได้อย่างสง่าผ่าเผย ไม่ต้องร้องขอพึ่งพิงให้คนรุ่นหลังคอยทำธุรกรรมประเคนให้ ลดปัญหากินแหนงแคลงใจในระดับหน่วยครอบครัว

\textbf{2. ผลกระทบด้านสุขภาพ (Health Impacts):}
สนับสนุนความเฟิร์มฝั่งจิตวิทยา ด้วยการมอบแพลตฟอร์มที่ไม่กดดัน ไม่ตะคอกเร่งเร้าให้ลูกค้าจำรหัสผ่านยากๆ ลดความเสี่ยงเป็นโรคซึมเศร้าจากการถูกมองว่าไร้ศักยภาพการใช้แอปฯ ไปพร้อมกันกับการถนอมดวงตาผู้สูงวัยจากจุดพิกเซลตัวอักษรที่แสนเป็นมิตร แถมลดความเสี่ยงหกล้มตามทางเดินฟุตบาทยามเดินทางไปซื้อสินค้า

\textbf{3. ผลกระทบด้านความปลอดภัย (Safety Impacts):}
เปลี่ยนผู้สูงอายุจากสถานะเหยื่อที่ขี่หลังเสือ ให้กลายเป็นผู้นำทางไอที ด้วยโครงสร้างระบบที่เราเขียนล็อกดาวน์เส้นทางการรั่วไหล (Secure by Design Approach) \cite{pressman2014} ทำลายรอยต่อทิ้งไม่ให้ลิงก์มิจฉาชีพแทรกซอนเข้ามาก่อความเสียหายสูบดึงเงินออมได้ง่ายๆ

\textbf{4. ผลกระทบด้านกฎหมาย (Legal Impacts):}
เคารพตามข้อบังคับกติกาเครือรัฐ (PDPA) อย่างเคร่งครัดเหนือสิ่งอื่นใด ข้อตกลง Consent ขออนุญาตเผยแพร่ข้อมูลในแพลตฟอร์มถูกรีดให้สั้น ตรงเป้า ชัดแจ้ง ไม่ใช้ภาษาพรรณนาลวงตาเพื่อฉกฉวยช่องโหว่ทางกฎหมายรังแกผู้บริโภคที่รู้เท่าไม่ถึงการณ์

\textbf{5. ผลกระทบด้านวัฒนธรรม (Cultural Impacts):}
คือการประกาศกร้าวในเรื่องการโอบรัดความแตกต่างระหว่างวัย (Age-Inclusive Culture) แสดงให้เห็นว่าไม่ระบุบอายุคนเล่นก็คู่ควรกับการรันเทคโนโลยีขั้นสูงสุด นำพาสังคมไทยให้ตระหนักรู้เบิกบานรับวัฒนธรรมที่ลูกหลานและตายายส่งยิ้มคุยเรื่องชอปปิงแบบไม่อายใครในศักราชเทคโนโลยียุคเจเนอเรชันถัดไป (Cultural Paradigm Shift)
